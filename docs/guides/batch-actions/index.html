Batch Actions

We've enhanced the batch_action feature to make it more customizable and extendable. We only list the adjustments we've made in the sections below. Please visit ActiveAdmin for basic usage guides.

Inputs#

caution

The following input types are currently not supported:

  • :file
  • :rich_text_area
info

We've updated input type namings to match with Formtastic inputs for consistency.

  • :text -> :string
  • :textarea -> :text
  • :checkbox -> :boolean
  • Implied
  • Explicit
e.g., app/admin/users.rb
ActiveAdmin.register User do  batch_action :flag, form: {    reason: :string,    notes:  :text,    hide:   :boolean,  } do |ids, inputs|    # ...  endend
e.g., app/admin/users.rb
ActiveAdmin.register User do  batch_action :flag, form: {    reason: {as: :string, label: "What's the reason?"},    notes: {as: :text, input_html: {value: "custom-class"}},    hide: {as: :switch}  } do |ids, inputs|    ...  endend

Confirm Dialog#

You can specify the configurations for the confirm dialogue by passing in the confirm_dialog option. Visit the confirm dialog guide for more information.

e.g., app/admin/users.rb
ActiveAdmin.register User do  batch_action :flag, confirm_dialog: {    buttons: {      ok: {class: "btn btn-danger", text: "Confirm!!!"}    }  } do |ids, inputs|    ...  endend

You can override the default configurations by following the instructions below:

  • CSS Class
  • Translation
Create a css_claases.yml and specify the changes. Visit the configuration guide for more information.

e.g., config/adminterface/admin/css_classes.yml
---table_tools:  confirm_dialog:    cancel: btn btn-link    ok: btn btn-primary
Create an active_admin.en.yml file and specify the keys you want to update.

config/locales/active_admin.en.yml
---en:  active_admin:    batch_actions:      confirm_dialog:        cancel: Cancel        ok: Okay!
