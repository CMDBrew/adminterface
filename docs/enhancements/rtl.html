RTL

There are about 12 languages written from right to left (e.g., Arabic, Hebrew, etc.) with more than 300 million native speakers. Learn how to enable support for right-to-left text in Adminterface to support these users.

Get Familiar#

Adminterface uses Bootstrap, so the RTL support is inherited from Bootstrap using the RTLCSS project.

Configuration#

To change the language direction to RTL, you can specify the lang_dir configuration at different levels.

OptionTypeDefaultDescription
lang_dirString, Symbol, or ProcltrChange the dir attribute for html.
  • Global
  • Namespace
  • Resource
This will change the default configurations globally for the entire app.

config/initializers/active_admin.rb
ActiveAdmin.setup do  # Options 1: Set a fixed value  config.lang_dir = "rtl"
  # Options 2: Use a proc  config.lang_dir = proc { !!params[:rtl] ? "rtl" : "ltr" }end
This will change the default configurations for the specified namespace.

config/initializers/active_admin.rb
ActiveAdmin.setup do  config.namespace :site_1 do |site|    # Options 1: Set a fixed value    site.lang_dir = "rtl"
    # Options 2: Use a proc    site.lang_dir = proc { !!params[:rtl] ? "rtl" : "ltr" }  endend
This will only change the configurations for this resource.

app/admin/users.rb
ActiveAdmin.register User do  # Options 1: Set a fixed value  config.lang_dir = "rtl"
  # Options 2: Use a proc  config.lang_dir = proc { !!params[:rtl] ? "rtl" : "ltr" }end

Implementation#

Requirements#

  1. Make sure you are using webpacker ~> 5.4.3 and run bundle install

    Gemfile
    gem "webpacker", "~> 5.4.3"

  2. Upgrade postcss NPM packages:

    # upgrade webpacker to 5.4.3$ yarn upgrade @rails/webpacker@5.4.3
# upgrade postcss$ yarn add postcss@^8.3.11
# upgrade postcss-loader$ yarn add postcss-loader@4.2.0

  3. Hotfix postcss loader error for PostCSS 8 (rails/webpacker/issues#2784)

    config/webpack/environment.js
    const { environment } = require('@rails/webpacker')
environment.loaders.keys().forEach(loaderName => {  const loader = environment.loaders.get(loaderName)  loader.use.forEach(loaderConfig => {    if (loaderConfig.options && loaderConfig.options.config) {      loaderConfig.options.postcssOptions = loaderConfig.options.config      delete loaderConfig.options.config    }  })})

RTL only#

  1. Install rtlcss

    $ yarn add rtlcss

  2. Add to rtlcss to postcss.config.js

    postcss.config.js
    module.exports = {  plugins: [    require('rtlcss'),    ...  ]}

  3. Set the entire application to RTL

    config/initializers/active_admin.rb
    ActiveAdmin.setup do  config.lang_dir = "rtl"end

LTR and RTL at the same time#

caution

You can enable both RTL and LTR in your project. However, please be aware that loading both files can be a real performance bottleneck.

  1. Install postcss-rtlcss

    $ yarn add postcss-rtlcss

  2. Add to postcss-rtlcss to postcss.config.js

    postcss.config.js
    module.exports = {  plugins: [    require('postcss-rtlcss'),    ...  ]}

  3. Now you can follow the instructions in Configuration to configure your application
