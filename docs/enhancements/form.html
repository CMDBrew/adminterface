Form Page

Adminterface inherits the functionalities from ActiveAdmin and applies Bootstrap styles. In addition, we've extended the supported input types and made some enhancements to existing features.

Below is a basic example of the form page. Please visit ActiveAdmin for usage guides.

ActiveAdmin.register Post do  form do |f|    f.input :title    ...    f.actions  endend

Multiple forms#

app/admin/users.rb
ActiveAdmin.register User do  form multiple: true do    form_section [:admin, resource] do |f|      f.input :name      ...      f.actions    end
    form_section [:admin, resource] do |f|      f.input :email      ...      f.actions    end  endend

Nested forms#

We've replaced jQuery with SortableJS. However, nested forms work the same as ActiveAdmin.

app/admin/posts.rb
ActiveAdmin.register Post do  form do |f|    ...
    f.inputs do      f.has_many :taggings, sortable: :position, sortable_start: 1 do |t|        t.input :tag      end    end
    ...  endend

The has_many fields do not support Arbre elements natively. See Github issue #4043. Here are a couple of workaround examples:

  • Columns
  • Arbre
app/admin/users.rb
ActiveAdmin.register User do  f.has_many :user_addresses, allow_destroy: true, sortable: :position, sortable_start: 1 do |k|    k.inputs(class: "row") do      k.input :fullname, wrapper_html: {class: "col-lg-6"}    end +      k.inputs(class: "row") do        k.input :address_line1, wrapper_html: {class: "col-lg-6"}        k.input :address_line2, wrapper_html: {class: "col-lg-6"}      end  endend
app/admin/users.rb
ActiveAdmin.register User do  f.has_many :user_addresses, allow_destroy: true, sortable: :position, sortable_start: 1 do |k|    k.inputs(class: "row") do      k.input :fullname, wrapper_html: {class: "col-lg-6"}    end +      Arbre::Context.new do        para "some hint texts"      end.to_s +      k.inputs(class: "row") do        k.input :address_line1, wrapper_html: {class: "col-lg-6"}        k.input :address_line2, wrapper_html: {class: "col-lg-6"}      end  endend

Inputs#

Formtastic's input helper helps you create input fields quickly. Here is an example output using the input as string helper:

tip

To change the HTML options for the elements you can use

  • wrapper_html: for <div class="input ...">
  • label_html: for <label>
  • input_html: for <input>
  • Ruby
  • HTML
f.input :name, as: :string
<!-- wrapper (BEGIN) --><div class="string input required mb-3 stringish" id="user_name_input">  <!-- label (BEGIN) -->  <label for="user_name" class="label form-label">Name<abbr title="required">*</abbr></label>  <!-- label (END) -->
  <!-- input (BEGIN) -->  <input id="user_name" class="form-control" type="text" value="Cyndi Fay" name="user[name]">  <!-- input (END) --></div><!-- wrapper (END) -->

Boolean#

:boolean - a checkbox. Default for column types: :boolean.

  • Configuration: Formtastic
  • Styling: Bootstrap
f.input :accept, as: :boolean

CheckBoxes#

:check_boxes - a set of checkbox inputs. Alternative to :select for ActiveRecord-associations: has_many, and has_and_belongs_to_many.

  • Configuration: Formtastic
  • Styling: Bootstrap
f.input :interests, as: :check_boxes, collection: %w[movies sports]

Inline Options#

You can pass in inline: true to render the options inline.

f.input :interests, as: :check_boxes, collection: %w[movies sports], inline: true

Color#

:color - a color select using the Bootstrap color component. New in HTML5.

  • Configuration: Formtastic
  • Styling: Bootstrap
f.input :favorite_color, as: :color

Input Group#

Use the append and prepend options. When you pass in either of them, it will wrap the input inside the Bootstrap's input-group element.

  • Prepend
  • Append
f.input :favorite_color, as: :color, prepend: content_tag(:span, "some text", class: "input-group-text")
f.input :favorite_color, as: :color, append: content_tag(:span, "some text", class: "input-group-text")

Country#

:country - a select menu of country names. Default for column types: :string with name "country".

Plugin Required

Requires a country_select gem to be installed.

Compatibility

BatchAction form only supports:

  • :priority_countries
  • :except
  • :only
  • Configuration: Formtastic, CountrySelect
  • Styling: Bootstrap
  • Basic
  • Prioritize
  • Exclude
  • Only
f.input :country, as: :country
f.input :country, as: :country, priority_countries: ["GB", "FR", "DE"]
f.input :country, as: :country, except: ["GB", "FR", "DE"]
f.input :country, as: :country, only: ["GB", "FR", "DE"]

Input Group#

Use the append and prepend options. When you pass in either of them, it will wrap the input inside the Bootstrap's input-group element.

  • Prepend
  • Append
f.input :country, as: :country, prepend: content_tag(:span, "some text", class: "input-group-text")
f.input :country, as: :country, append: content_tag(:span, "some text", class: "input-group-text")
