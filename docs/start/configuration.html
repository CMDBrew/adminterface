Configuration

Adminterface extends the configurations from ActiveAdmin to make it easier to customize the styles and behaviours of your application.

CSS Classes#

Customize the default CSS classes for ActiveAdmin's components. You can view Bootstrap CSS classes here for more references.

  • Make sure you add the proper indentations while customizing the CSS classes.

This will change the default configurations globally for the entire app.

config/initializers/active_admin.rb
ActiveAdmin.setup do  config.css_classes = {    header: {      wrapper: "navbar-dark bg-dark navbar-expand-lg"    }  }end
This will change the default configurations for the specified namespace.

config/initializers/active_admin.rb
ActiveAdmin.setup do  config.namespace :site_1 do |site|    site.css_classes = {      header: {        wrapper: "navbar-dark bg-dark navbar-expand-lg"      }    }  endend
This will only change the configurations for this resource.

app/admin/users.rb
ActiveAdmin.register User do  config.css_classes = {    header: {      wrapper: "navbar-dark bg-dark navbar-expand-lg"    }  }end

You can also load the configs from YAML files to keep things tidy. For example, you can create a file

config/adminterface/admin/users/css_classes.yml
---action_items:  item: btn btn-secondaryheader:  container: container

And, load it inside your admin resource

app/admin/users.rb
ActiveAdmin.register User do  config.css_classes = YAML.load_file(Rails.root.join("config/adminterface/admin/users/css_classes.yml").to_s)end

ActionItems#

action_items — the actions items for the resource. Available actions items are :new, :edit, and :destroy.

OptionTypeDescription
css_classStringChange the CSS class for this button. If not set, it will default to the action_items.item setting in css_classes.yml
icon_classStringAdd the classes for the icon. The based class is set globally using icon setting in css_classes.yml, which defaults to Bootstrap Icons
onlyArrayThe controller actions you want this action_item to display

Columns#

columns — the Columns component.

OptionTypeDescription
breakpointStringUse Bootstrap's breakpoints. Available options are sm, md, lg, xl, xxl or null

Filter#

filter — filters on the index page.

OptionTypeDescription
positionStringAvailable options are sidebar, table_tools, or aside

Flash#

flash — the flash messages for the app.

OptionTypeDescription

Header#

header — the Header component.

OptionTypeDescription
positionStringAvailble options are top, sidebar, or aside
css_class.top, css_class.sidebar, css_class.asideStringposition specific css classes

We provide three default navigation layouts defined using the CSS grid, and the layout is specified using the .layout-navigation-#{TYPE} class.

You can add your custom layout by adding custom CSS styles. Here is an example of creating a bottom navigation layout.

First, create a stylesheet and add styles to the custom navigation.

app/javascript/stylesheets/adminterface/layouts/_navigation_bottom.scss
$_header-height: 64px;$_header-zindex: $zindex-fixed;$_title-bar: $zindex-fixed - 1;$_title-bar-height: 54px;$_title-bar-zindex: $zindex-fixed - 1;
body.layout-navigation-bottom {  grid-template-areas:    "title_bar"    "content"    "header";  display: grid;  grid-template-columns: minmax(0, 1fr);  grid-template-rows: $_title-bar-height minmax(0, 1fr) minmax($_header-height, auto);
  #header {    grid-area: header;    overflow-y: visible;    z-index: $_header-zindex;  }
  #title_bar {    grid-area: title_bar;    z-index: $_title-bar-zindex;  }
  #wrapper { grid-area: content; }
  @include media-breakpoint-up($aside-breakpoint) {    #filters.aside { top: calc(#{$_header-height} + #{$_title-bar-height}); }  }}

Import the custom stylesheet inside active_admin.scss.

app/javascript/stylesheets/active_admin.scss
@import "~@cmdbrew/adminterface/src/scss/base";@import "adminterface/layouts/navigation_bottom"

Finally, specify the configuration for this custom navigation layout for the entire app, a namespace, or a specific resource the same way as before.

config/adminterface/admin/users/components.yml
---header:  position: bottom  css_class:    bottom: navbar-dark bg-dark navbar-expand-lg

Inputs#

inputs: — formtastic and JavaScript input components.

You can add configurations for any available input types. You can find the default input types inside the Form Page guide.

Plugins are defined inside the js as an array of key and value pairs. Please see the JavaScript guide for the default plugins that comes with Adminterface.

inputs:  date_picker:    js:      - name: flatpickr        default: true        options:          altFormat: Y-m-d          enableTime: false
OptionTypeDescription
nameStringThe name of the plugin
defaultBooleanWhether to enable this plugin by default
optionskey: value pairsVendor specific configurations for the plugin
tip

It is also possible to add custom input types. For example:

inputs:  my_custom_input:    ...

Please see the official Formtastic guide and our JavaScript guide for more information.

Theme#

Adminterface provides a list of variables that you can use to customize the theme for your application. To customize the theme of your Adminterface application, you can:

  1. Create a stylesheet that holds all the variables that you want to overwrite.

    app/javascript/stylesheets/_adminterface_variables.scss
    $primary: #8E6DBF;...

  2. Import the stylesheet inside the active_admin.scss file.

    app/javascript/stylesheets/active_admin.scss
    @import "adminterface_variables";@import "~@cmdbrew/adminterface/src/scss/base";
    tip

    The line @import "adminterface_variables"; has to be before the line that imports our stylesheets @imoprt "~@cmdbrew/adminterface/…"; You can check the official Sass guide to see how variable overwrites work.
