Configuration

We've kept existing functionalities and extended them to make it easier for you to customize your application. Please see ActiveAdmin for basic configurations and usage guides.

Scoping#

tip

You can load your configs from YAML files to keep things tidy.

e.g., app/admin/users.rb
ActiveAdmin.register User do  config.css_classes = YAML.load_file(Rails.root.join("config/active_admin_bootstrap/admin/users/css_classes.yml").to_s)  config.components = YAML.load_file(Rails.root.join("config/active_admin_bootstrap/admin/users/components.yml").to_s)end
  • Global
  • Namespace
  • Resource
This will change the default configurations globally for the entire app.

config/initializers/active_admin.rb
ActiveAdmin.setup do  config.css_classes = {header: {wrapper: "navbar-dark bg-dark navbar-expand-lg"}}  config.components = {action_items: {new: {icon_class: "bi-plus"}}}end
This will change the default configurations for the specified namespace.

config/initializers/active_admin.rb
ActiveAdmin.setup do  config.namespace :site_1 do |site|    site.css_classes = {header: {wrapper: "navbar-dark bg-dark navbar-expand-lg"}}    site.components = {action_items: {new: {icon_class: "bi-plus"}}}  endend
This will only change the configurations for this resource.

e.g., app/admin/users.rb
ActiveAdmin.register User do  config.css_classes = {header: {wrapper: "navbar-dark bg-dark navbar-expand-lg"}}  config.components = {action_items: {new: {icon_class: "bi-plus"}}}end

ActionItems#

action_items — the actions items for the resource. Available actions items are :new, :edit, and :destroy.

OptionTypeDescription
css_classStringChange the CSS class for this button. If not set, it will default to the action_items.item setting in css_classes.yml
icon_classStringAdd the classes for the icon. The based class is set globally using icon setting in css_classes.yml, which defaults to Bootstrap Icons
onlyArrayThe controller actions you want this action_item to display

Comments#

comments — the ActiveAdminComment component.

OptionTypeDescription
inputStringAvailable Options: rich_text or text

Columns#

columns — the Columns component.

OptionTypeDescription
breakpointStringUse Bootstrap's breakpoints. Available options are sm, md, lg, xl, xxl or null

Filter#

filter — filters on the index page.

OptionTypeDescription
positionStringAvailable options are sidebar, table_tools, or aside

Flash#

flash — the flash messages for the app.

OptionTypeDescription
dismissibleBooleanSet to true if you want to display the dismiss button

Header#

header — the Header component.

OptionTypeDescription
positionStringAvailble options are top, sidebar, or aside
css_class.top, css_class.sidebar, css_class.asideStringposition specific css classes

IndexAsGrid#

index_as_grid: - the IndexAsGrid component.

OptionTypeDescription
breakpointStringUse Bootstrap's breakpoints. Available options are sm, md, lg, xl, xxl or null

Inputs#

inputs: — formtastic input components

OptionTypeDescription
date_pickerNested key, value pairsSee Flatpickr for config options.
datetime_pickerNested key, value pairsSee Flatpickr for config options.
tom_selectNested key, value pairsSee TomSelect for config options.
text.counter.typeStringThe mode for counting. Available options are letter, word
text.counter.limitIntegerMaximum count

Sidebar#

sidebar: — filters on the index page.

OptionTypeDescription
positionStringThe default placement for the sidebar. Available options are left, or right
