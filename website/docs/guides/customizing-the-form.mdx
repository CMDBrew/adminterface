# Customizing the Form

````mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import CodeBlock from '@theme/CodeBlock';
````

We've kept the existing functionalities and applied [Bootstrap] styles with some additional inputs types and enhancements.

## Form
````mdx-code-block
<Tabs
  defaultValue="basic"
  values={[
    {label: 'Basic', value: 'basic'},
    {label: 'Multiple', value: 'multiple'},
  ]}>

  <TabItem value="basic">

  See [ActiveAdmin] for more information.

  </TabItem>
  <TabItem value="multiple">

  ```ruby title="e.g., app/admin/users.rb" {2-3,9}
  ActiveAdmin.register User do
    form multiple: true do
      form_section [:admin, resource] do |f|
        f.input :name
        ...
        f.actions
      end

      form_section [:admin, resource] do |f|
        f.input :email
        ...
        f.actions
      end
    end
  end
  ```

  </TabItem>
</Tabs>
````

## Inputs
[Formtastic]'s `f.input` helps you create input fields quickly. Here is an example output using the `input` as `string` helper:

:::tip
To change the HTML options for the elements you can use
- wrapper_html: for `<div class="input ...">`
- label_html: for `<label>`
- input_html: for `<input>`
:::

````mdx-code-block
<Tabs
  defaultValue="ruby"
  values={[
    {label: 'Ruby', value: 'ruby'},
    {label: 'HTML', value: 'html'},
  ]}>

  <TabItem value="ruby">

  ```ruby
  f.input :name, as: :string
  ```

  </TabItem>
  <TabItem value="html">

  ```html
  <!-- wrapper (BEGIN) -->
  <div class="string input required mb-3 stringish" id="user_name_input">
    <!-- label (BEGIN) -->
    <label for="user_name" class="label form-label">Name<abbr title="required">*</abbr></label>
    <!-- label (END) -->

    <!-- input (BEGIN) -->
    <input id="user_name" class="form-control" type="text" value="Cyndi Fay" name="user[name]">
    <!-- input (END) -->
  </div>
  <!-- wrapper (END) -->
  ```

  </TabItem>
</Tabs>
````


### Boolean
`:boolean` - a checkbox. Default for column types: `:boolean`.
- **Configuration**: [Formtastic/BooleanInput]
- **Styling**: [Bootstrap/checks-radios#checks]

```ruby
f.input :accept, as: :boolean
```

### CheckBoxes
`:check_boxes` - a set of check_box inputs. Alternative to `:select` for ActiveRecord-associations: `has_many`, and `has_and_belongs_to_many`.
- **Configuration**: [Formtastic/CheckBoxesInput]
- **Styling**: [Bootstrap/checks-radios#checks]

:::info Works with

- [Inline Options](#inline-radio--check-options)

:::

```ruby
f.input :interests, as: :check_boxes, collection: %w[movies sports]
```

### Color
`:color` - a color select using the [Bootstrap] color component. New in HTML5.
- **Configuration**: [Formtastic/StringInput]
- **Styling**: [Bootstrap/form-control#color]

:::info Works with

- Works with: [Input Group](#input-group)

:::

```ruby
f.input :favorite_color, as: :color
```

### Country
`:country` - a select menu of country names. Default for column types: :string with name "country".
- **Configuration**: [Formtastic/CountryInput], [CountrySelect]
- **Styling**: [Bootstrap/select]

:::info Works with

- [Input Group](#input-group)

:::

:::caution Plugin Required

Requires a `country_select` gem to be installed.

:::

:::caution Compatibility

BatchAction form only supports:
- `priority_countries`
- `except`
- `only`

:::

````mdx-code-block
<Tabs
  defaultValue="basic"
  values={[
    {label: 'Basic', value: 'basic'},
    {label: 'Prioritize', value: 'prioritize'},
    {label: 'Exclude', value: 'exclude'},
    {label: 'Only', value: 'only'},
  ]}>

  <TabItem value="basic">

  ```ruby
  f.input :country, as: :country
  ```

  </TabItem>
  <TabItem value="prioritize">

  ```ruby
  f.input :country, as: :country, priority_countries: ["GB", "FR", "DE"]
  ```

  </TabItem>
  <TabItem value="exclude">

  ```ruby
  f.input :country, as: :country, except: ["GB", "FR", "DE"]
  ```

  </TabItem>
  <TabItem value="only">

  ```ruby
  f.input :country, as: :country, only: ["GB", "FR", "DE"]
  ```

  </TabItem>
</Tabs>
````

## Addons
:::info

Check the [Inputs](#inputs) section for inputs work with the addons.

:::

### Input Group
We've added support for [Bootstrap/input-group]. Use the `:append` and `:prepend` options to add html before and after the input. When you pass in either of the options it will wrap the input inside the `.input-group` element.

```ruby
f.input :name, as: :string,
  append: content_tag(:span, "append text", class: "input-group-text"),
  prepend: content_tag(:span, "prepend text", class: "input-group-text")
```

### Inline Options
You can pass in the `inline: true` option if you want to render the options inline.

```ruby
f.input :interests, as: :check_boxes, collection: %w[movies sports], inline: true
f.input :interests, as: :radio, collection: %w[movies sports], inline: true
```

### Counter
You can enable counter by passing the `counter:` option. See [Configuration](/docs/start/configuration#inputs) for more information.

:::caution

To ensure data integrity, you should also implement server-side validations.

:::

````mdx-code-block
<Tabs
  defaultValue="basic"
  values={[
    {label: 'Basic', value: 'basic'},
    {label: 'Options', value: 'options'},
  ]}>

  <TabItem value="basic">

  ```ruby
  f.input :biography, as: :text, counter: true
  ```

  </TabItem>
  <TabItem value="options">

  ```ruby
  f.input :biography, as: :text, counter: {type: "word", limit: 10}
  ```

  </TabItem>
</Tabs>
````

## Nested Resources
:::tip

We've replaced jQuery sortable with [SortableJS]

:::

Nested forms work exactly the same as [ActiveAdmin]. You can find the docs [here](https://activeadmin.info/5-forms.html#nested-resources).

```ruby title="e.g., app/admin/posts.rb" {6-8}
ActiveAdmin.register Post do
  form do |f|
    ...

    f.inputs do
      f.has_many :taggings, sortable: :position, sortable_start: 1 do |t|
        t.input :tag
      end
    end

    ...
  end
end
```

### HasMany & Arbre
:::danger

The has_many fields do not work support Arbre elements since it doesn't have access to them. See [#4043](https://github.com/activeadmin/activeadmin/issues/4043).

:::

To add columns to your nested form:

```ruby title="e.g., app/admin/users.rb" {3-4,6-8}
ActiveAdmin.register User do
  f.has_many :user_addresses, allow_destroy: true, sortable: :position, sortable_start: 1 do |k|
    k.inputs(class: "row") do
      k.input :fullname, wrapper_html: {class: "col-lg-6"}
    end +
      k.inputs(class: "row") do
        k.input :address_line1, wrapper_html: {class: "col-lg-6"}
        k.input :address_line2, wrapper_html: {class: "col-lg-6"}
      end
  end
end
```

To add Arbre elements to your nested form:
```ruby title="e.g., app/admin/users.rb" {6-8}
ActiveAdmin.register User do
  f.has_many :user_addresses, allow_destroy: true, sortable: :position, sortable_start: 1 do |k|
    k.inputs(class: "row") do
      k.input :fullname, wrapper_html: {class: "col-lg-6"}
    end +
      Arbre::Context.new do
        para "some hint texts"
      end.to_s +
      k.inputs(class: "row") do
        k.input :address_line1, wrapper_html: {class: "col-lg-6"}
        k.input :address_line2, wrapper_html: {class: "col-lg-6"}
      end
  end
end
```
