# Arbre Components

````mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import CodeBlock from '@theme/CodeBlock';
````

:::caution

These components works inside `rb` or `arb` file within the ActiveAdmin scope.

:::

We've kept the existing functionalities and applied [Bootstrap] styles with some enhancements. Please see [ActiveAdmin] for basic configurations.

## Icons
We've added a helper for rendering icons inside [ActiveAdmin]. The default icons we're using are [Bootstrap Icons].
If you wish, you can change the default icon class like below:
```yml title="config/active_admin_bootstrap/.../css_classes.yml" {2}
---
icon: your-custom-class
```

To render the icon inside Arbre, call:
```ruby
aa_icon("bi-alarm")
```

## Panels
See basic configuration options in [ActiveAdmin/arbre-components/panels].

### Title not required
We've removed the requirements for the `title` param. If you don't pass in the title param or passed in an empty value, the panel header will not be visible.

```ruby {1,5}
panel do
  h1 "Hello world"
end

panel "" do
  h1 "Hello world"
end
```

### HTML Options
You can pass in `:header_html` and `:body_html` to add additional attributes to the panel's header and body, respectively.

```ruby {1}
panel header_html: {class: "my-header-class"}, body_html: {class: "my-body-class"} do
  h1 "Hello world"
end
```

## Columns
We've replaced columns from using inline css to using [Bootstrap/layout/grid]. See basic configuration options in [ActiveAdmin/arbre-components/columns].

### Gutter
You can change the gutter between columns by passing in your own gutter class.

````mdx-code-block
<Tabs
  defaultValue="input"
  values={[
    {label: 'Ruby', value: 'input'},
    {label: 'HTML', value: 'output'},
  ]}>

  <TabItem value="input">

  ```ruby {1}
  columns(class: "g-5") do
    ...
  end
  ```

  </TabItem>
  <TabItem value="output">

  ```html {1}
  <div class="g-5 row">
    ...
  </div>
  ```

  </TabItem>
</Tabs>
````

### Sizing
:::caution

`:max_width` and `:min_width` options no longer works.

:::

````mdx-code-block
<Tabs
  defaultValue="input"
  values={[
    {label: 'Ruby', value: 'input'},
    {label: 'HTML', value: 'output'},
  ]}>

  <TabItem value="input">

  ```ruby {2,6}
  columns do
    column(class: "custom-column", size: "md", span: 8) do
      ...
    end

    column(class: "custom-column", size: "md", span: 4) do
      ...
    end
  end
  ```

  </TabItem>
  <TabItem value="output">

  ```html {2,6}
  <div class="g-3 row">
    <div class="col-md-8 custom-column">
      ...
    </div>

    <div class="col-md-4 custom-column">
      ...
    </div>
  </div>
  ```

  </TabItem>
</Tabs>
````

## Table For
See basic configuration options in [ActiveAdmin/arbre-components/table_for].

### Overflow & Scrolling
You can add scrolling to your table by usings [Bootstrap]'s `.table-responsive` class.

```ruby {1}
div class: "table-responsive" do
  table_for order.payments do
    ...
  end
end
```

## Tabs
:::tip

We've replaced jQuery tabs with [Bootstrap].

:::

See basic configuration options in [ActiveAdmin/arbre-components/tabs].

### HTML Options
You can pass in `:nav_html` and `:content_html` to add additional attributes to the tab's nav and content, respectively.

```ruby {1}
tabs nav_html: {class: "my-tabs-nav"}, content_html: {class: "my-tabs-content"} do
  tab "tab-1" { ... }
  tab "tab-2" { ... }
end
```

### HTTP Tabs
Sometimes, you would want the page to refresh when viewing another tab. To achieve that, you can pass in `http: true` to the tabs.
If you add an `:id` to the tab, it will automatically set as the anchor for the nav links.

```ruby {1}
tabs(http: true, id: "test") do
  tab "tab-1" { ... }
  tab "tab-2" { ... }
end
```

The tab parameter is memorized in the session, so it works with form submission as well! ✌️
```ruby title="e.g., app/admin/users.rb" {4}
ActiveAdmin.register User do
  form do |f|
    panel do
      tabs http: true, id: "user-form" do
        tab :info do
          f.inputs do
            f.input :name
          end
        end
        tab :password do
          f.inputs do
            f.input :password
          end
        end
      end

      f.actions
    end
  end
end
```
