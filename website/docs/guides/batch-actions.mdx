# Batch Actions

````mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import CodeBlock from '@theme/CodeBlock';
````

We've enhanced the `batch_action` feature to make it more customizable and extendable. We only list the adjustments we've made in the sections below. Please visit [ActiveAdmin] for basic configurations and usage guides.

## Inputs
:::caution

The following input types are currently not supported:
- `:file`
- `:rich_text_area`

:::

:::info

We've updated input type namings to match with Formtastic inputs for consistency.
- `:text` -> `:string`
- `:textarea` -> `:text`
- `:checkbox` -> `:boolean`

:::


````mdx-code-block
<Tabs
  defaultValue="basic"
  values={[
    {label: 'Basic', value: 'basic'},

    batch_action :flag, form: {
      reason: :string,
      notes:  :text,
      hide:   :boolean,
    } do |ids, inputs|
      # ...
    end
  end
  ```

  <TabItem value="explicit">

  ActiveAdmin.register User do
    batch_action :flag, form: {
      reason: {as: :string},
      notes: {as: :text, input_html: {value: "custom-class"}},
      hide: {as: :switch}
    } do |ids, inputs|
      ...
    end
  end
  ```

  </TabItem>
</Tabs>
````

## Confirm Dialog
You can specify the configurations for the confirm dialogue by passing in the `confirm_dialog` option. See the [Confirm Dialog] guide for more information.

```ruby title="e.g., app/admin/users.rb" {2-5}
ActiveAdmin.register User do
  batch_action :flag, confirm_dialog: {
    buttons: {
      ok: {class: "btn btn-danger", text: "Confirm!!!"}
    }
  } do |ids, inputs|
    ...
  end
end
```

### Override the Defaults
````mdx-code-block
import ConfirmCssClass from '!!raw-loader!@site/../config/locales/active_admin.en.yml';

<Tabs
  defaultValue="css_class"
  values={[
    {label: 'CSS Class', value: 'css_class'},
    {label: 'Translation', value: 'translation'},
  ]}>

  <TabItem value="css_class">
  To update the default CSS classes, you can create a <code>css_claases.yml</code> and specify the changes. See <a href="/docs/start/configuration#css-classes">Configuration</a> for more information.
  <br/>
  <br/>

  ```yaml title="e.g., config/active_admin_bootstrap/admin/css_classes.yml" {2-6}
  ---
  table_tools:
    confirm_dialog:
      cancel: btn btn-link
      ok: btn btn-primary
  ```

  </TabItem>
  <TabItem value="translation">
  To update the translations, you can create an <code>active_admin.en.yml</code> file and specify the keys you want to update.
  <br/>
  <br/>

  ```yaml title="config/locales/active_admin.en.yml" {4-7}
  ---
  en:
    active_admin:
      batch_actions:
        confirm_dialog:
          cancel: Cancel
          ok: Okay!
  ```
  </TabItem>
</Tabs>
````
